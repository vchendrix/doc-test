Package Service API 1.2.0¶

GET /packages/{identifier}¶

Get a data package by its identifier

Get a data package that you have permissions to edit

Parameters

  • identifier (string) – the ESS-DIVE unique identifier (e.g ess-dive-XXXXXXX-2018…, doi:10.xxxx/xxxxx)

Status Codes

  • 200 OK – List a Single data package

  • 400 Bad Request – Bad Request

  • 401 Unauthorized – Unauthorized Access

  • 404 Not Found – Not Found

  • 405 Method Not Allowed – Method Not Allowed

  • 500 Internal Server Error – Internal Server Error

GET /packages¶

List data packages

List all of the data packages that you have permissions to edit.

Query Parameters

  • page_size (integer) – The number of data packages to return per request.

  • row_start (integer) – The row number to start on . Use this for paging results

Status Codes

  • 200 OK – lists the data packages submitted by the user

  • 400 Bad Request – Bad Request

  • 401 Unauthorized – Unauthorized Access

  • 404 Not Found – Not Found

  • 405 Method Not Allowed – Method Not Allowed

  • 500 Internal Server Error – Internal Server Error

POST /packages¶

Create a data package

Create a new JSON-LD representation of a data package with data files (optional).

Status Codes

  • 201 Created – Package successfully created

  • 400 Bad Request – Bad Request. There are validation errors

  • 401 Unauthorized – Unauthorized Access

  • 405 Method Not Allowed – Method Not Allowed

  • 500 Internal Server Error – Internal Server Error

PUT /packages¶

Update a data package

Update a JSON-LD representation of a data package with data files (optional).

Status Codes

  • 201 Created – Package successfully created

  • 400 Bad Request – Bad Request. There are validation errors

  • 401 Unauthorized – Unauthorized Access

  • 405 Method Not Allowed – Method Not Allowed

  • 500 Internal Server Error – Internal Server Error
